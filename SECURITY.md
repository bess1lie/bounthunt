# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

Bountyhunt is a reconnaissance tool designed for **authorized bug bounty programs
only**. If you discover a security vulnerability in the tool itself, please report
it privately.

**Do not** open a public GitHub issue for security vulnerabilities.

Send details to: [bess1lie@users.noreply.github.com](mailto:bess1lie@users.noreply.github.com)

You should receive a response within 48 hours. If not, please follow up.

## Scope

- **In scope**: Remote code execution, privilege escalation, credential leakage,
  data corruption, or any issue that could compromise the system running
  bountyhunt.
- **Out of scope**: Missing features, false positives in scan results, or
  websites scanned without authorization (those are legal matters, not
  security vulnerabilities of this tool).

## Disclosure Policy

- We will acknowledge receipt of your report within 48 hours.
- We will provide an estimated timeline for a fix.
- We will notify you when the issue is resolved.
- We grant permission to publicly disclose the vulnerability 90 days after
  the fix is released.
